# AGENT 가이드

## 목적

이 레포지토리는 소규모 웹게임 포탈을 만드는 것을 목표로 합니다. AI 에이전트는 개발 방향과 설계 결정을 잃지 않도록 아래 규칙을 따르세요.

## 프로젝트 개요

- 백엔드: NestJS (TypeScript)
- 프론트엔드: React + Vite (TypeScript)
- 공통 타입: `packages/shared`

## 우선순위

1. 최소 동작 가능한 전체 경로(backend API ↔ frontend UI) 구현
2. 타입 안전성 유지 (공유 DTO 사용)
3. 테스트 가능한 로컬 개발 환경 제공
4. 점진적 기능 확장

## 개발 규칙

- 모든 변경은 관련 패키지 안에서만 수행합니다.
- 공통 타입은 `packages/shared`에 추가하고 버전 관리합니다.
- 가능한 경우 타입을 먼저 정의한 뒤 API와 UI를 구현합니다.
- 새로운 의존성 추가 시 루트 `package.json`의 워크스페이스 정책을 따릅니다.

## 코드 스타일

- TypeScript strict 모드 사용
- ESLint + Prettier 병행(설정 미비 시 우선 일관성 유지)

## 브랜치 & 커밋

- 기능별 브랜치: `feature/<짧은-설명>`
- 버그 수정: `fix/<짧은-설명>`
- 커밋 메시지는 유의미하게 작성

## 작업 흐름

- 작은 단위로 구현하고, 각 단계마다 로컬에서 `pnpm dev`로 확인
- QA 필요 시 간단한 수동 테스트 케이스 제공

## 추가 메모

- 실행 및 빌드 관련 명령은 각 패키지 `README.md`에 작성합니다.
- 에이전트는 변경 시 해당 변경 파일 목록과 간단한 실행 가이드를 남기세요.
